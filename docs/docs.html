DateFormatter dateformatter.js

Convert a date object into a more readable string based on a format the you specify. This method also extends the possiblity to use textual names for weekdays and months. At the moment these names are limited to English by default, however you can easly change the soruce code with your prefered language.

Example

var date = DateFormatter.format(new Date(), '%d %B %Y');
alert(date); // Ex. 01 January 2014

Methods

DateFormatter.format(date, format)

Paramters
date - JavaScript date object representing the date to format. Read more about the date object at MDN.
format - String with the format to insert the given time. See format information below.
Returns
String with the format replaced with the given date values.

Formats

Date

Format Description Example
%a Weekday, abbreviated.
%A Weeday, full name.
%w Weekday as a decimal number where 0 is Sunday and 6 is Saturday.
%d Day of the month as a zero-padded decimal number.
%b Month, abbreviated.
%B Month, full name.
%m Month as a zero-padded decimal number.
%y Year without century as a zero-padded decimal number.
%Y Year with century as a decimal number.

Time

Format Description Example
%H Hour (24-hour clock) as a zero-padded decimal number.
%I Hour (12-hour clock) as a zero-padded decimal number.
%p AM and PM
%M Minute as a zero-padded decimal number.
%S Second as a zero-padded decimal number.
%f Millisecond as a decimal number.

TimeFormatter timeformatter.js

Convert milliseconds into a more readable string based on a format the you specify.

Example

Basic

var time = TimeFormatter.format(126543210, '%%D %H %M %S %f');
alert(time); // 01 11 09 03 210

Others chars

The example above does not preduce a very readable result as there are no indications to what the numbers represents. We can fix this by entering other chars into the format.

TimeFormatter.format(126543210, '%%D day %H hours %M minutes %S seconds %f milliseconds');
// 01 day 11 hours 09 minutes 03 seconds 210 milliseconds

Stops

All formats up to days have either a single or double "%" as a prefix. A single prefix means that the value will "overflow" into the next bigger value. For example if you use a single prefix for minutes, %M, then the value will always be withing 0-60. However if double prefix is used then the value will not overflow but continue to count.

In the examples above you can see that the day has a double prefix while the rest that follow have a single. That is because I see the day as the last stop within that format. In most cases you will want that your biggest value has the double prefix and everything smaller a single.

TimeFormatter.format(126543210, '%%Hh %Mm %Ss');
// 35h 09m 03s
TimeFormatter.format(126543210, '%%M min %S sec');
// 2109 min 03 sec

Timer

A simple timer...

Example

Constructors

Timer()

Methods

start()

Start the timer.

stop()

Stop the timer.

reset()

Reset the start and stop time.

time()

Returns the number of milliseconds that has elapsed between start and stop was called respectively. If the timer is running, the currently elapsed time will be returned. If the timer has not been started '0' will be the value.

Returns
Elapsed milliseconds as an integer.
